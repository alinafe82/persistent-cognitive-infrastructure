# Code of Conduct

## Standard

Contributors are expected to keep discussion technical, respectful, and focused
on improving the project. Disagreement is acceptable. Harassment, threats,
targeted insults, doxxing, and repeated bad-faith disruption are not.

## Scope

This standard applies to project repositories, issue trackers, pull requests,
design discussions, release channels, and official community spaces.

## Enforcement

Maintainers may moderate discussions, close issues, block participants, or
restrict contribution rights when behavior harms project safety or integrity.
Security reports and private vulnerability handling remain governed by
SECURITY.md.
